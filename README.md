## Report Generation Tool "SPERMATOZOID"

Програмний засіб для автоматизованої генерації технічної та академічної звітності у форматах DOCX та PDF.

Утиліта використовує підхід "Docs as Code", дозволяючи формувати складні документи на основі Markdown-файлів, YAML-метаданих та XML-конфігурацій. Ключовою особливістю системи є можливість автоматичного виконання термінальних команд, зняття скріншотів консольного виводу (через `carbon-now`) та безшовна інтеграція вихідного коду безпосередньо у фінальний документ за допомогою шаблонізатора Velocity.

### Встановлення (Тільки для Windows)

1. Перейдіть на сторінку **[Releases](https://github.com/AndriyKramar2288/report-generation/releases)** на GitHub.

2. Завантажте останню версію інсталятора `reportsSetup.exe`.

3. Запустіть файл та пройдіть стандартний процес встановлення.

4. Готово! Програма автоматично додана до системної змінної `PATH`. Тепер ви можете відкрити будь-який термінал і викликати генератор командою `reports`.

### Основні можливості

- Автоматична підстановка скалярних змінних (ПІБ, група, номер роботи) у DOCX-шаблон.

- Інтерактивне виконання shell-скриптів із захопленням результату у вигляді зображень.

- Парсинг файлів вихідного коду за glob-паттернами та їх інтеграція у додатки звіту.

- Пакетна (каскадна) генерація масиву звітів з автоматичним розгортанням файлової структури та робочого середовища.

---

### Режими роботи

Застосунок підтримує два незалежні режими обробки даних: базовий та каскадний.

#### 1. Базовий режим (Одиничний звіт)

Призначений для перетворення одного конфігураційного файлу (`rom.md`) на готовий документ. Вимагає наявності файлу з YAML-метаданими та Markdown-розміткою у поточній директорії.

**Приклад використання:**

```bash
# Генерація звіту у форматах DOCX та PDF на основі файлу rom.md
reports -docx -pdf
```

Система зчитає параметри з `rom.md`, виконає необхідні команди, згенерує зображення у прихованому режимі та сформує вихідні файли на основі вбудованого шаблону.

#### 2. Каскадний режим (Пакетна обробка та AI-інтеграція)

Каскадний режим (`cascade`) дозволяє автоматизувати створення цілого курсу лабораторних робіт на основі єдиної XML-специфікації (`com.xml`). Цей режим оптимізовано для роботи у зв'язці з великими мовними моделями (LLM).

**Рекомендований робочий процес (AI Workflow):**

1. **Ініціалізація.** Запустіть каскадний режим у порожній директорії з прапорцем білда:

    ```bash
    reports cascade -b
    ```

   Оскільки файл `com.xml` відсутній, програма згенерує еталонний файл конфігурації, який містить готовий інженерний промпт (запит) для штучного інтелекту, і завершить роботу.

1. **Генерація контенту.** Відкрийте згенерований `com.xml`, скопіюйте вбудований промпт і відправте його до LLM (наприклад, ChatGPT, Claude, Gemini). Додайте до запиту свої персональні дані (ПІБ, групу) та файли методичних вказівок.

2. **Конфігурація.** Модель поверне готову XML-структуру, що містить вихідний код, логіку виконання термінальних команд та текст звітів. Вставте отриманий XML назад у файл `com.xml`, замінивши початковий промпт.

3. **Виконання.** Повторно запустіть ту саму команду:

    ```bash
    reports cascade -b
    ```

   Система розгорне робочі директорії для кожної роботи, створить файли коду, виконає термінальні команди, згенерує зображення результатів і збереже готові DOCX/PDF файли.
